# AshleyIQ — Speed-to-Lead Automation Module
## Claude Code Build Prompt · Conduit Next CRM

**Product:** AshleyIQ (powered by Conduit Next)
**Module:** Speed-to-Lead Automation
**Stack:** Next.js 14 App Router · Supabase · Clerk · Twilio · Resend · Claude Haiku 4.5 · Vercel
**Priority:** P0 — table stakes feature; first lender to respond wins the borrower

---

## PRIME DIRECTIVE

Build the Speed-to-Lead Automation module for AshleyIQ, the mortgage LO CRM inside Conduit Next. This module guarantees that every new lead — regardless of source — receives an intro SMS, intro email, and LO in-app push notification within **60 seconds** of entering the system. An escalation engine fires at 5 minutes, 10 minutes, and 30 minutes if the LO has not marked contact.

This is not a nice-to-have. Surefire CRM has it. Borrowers choose lenders within the first hour of inquiry. We must be first or we lose the deal.

## SECTION 1 — WHAT IT DOES (Full Behavior Spec)

### 1.1 Lead Ingestion Trigger

The speed-to-lead pipeline fires on **every new lead regardless of source**:

| Source | Entry Point |
|---|---|
| Point of Sale (POS) | Webhook from Encompass / Byte / Calyx |
| Zillow Premier Agent | `POST /api/webhooks/zillow` |
| Realtor.com | `POST /api/webhooks/realtorcom` |
| Manual entry (LO inputs form) | Lead create form, `POST /api/leads` |
| CSV bulk import | Background job, triggers per-lead on insert |
| Direct API | `POST /api/leads` from partner integrations |
| Landing page form | `POST /api/leads/inbound` |

On every insert into the `leads` table (via Supabase trigger + Edge Function), the pipeline begins immediately.

### 1.2 The 60-Second Response Sequence

When a lead is created:

1. **Routing** (0–2 seconds): `POST /api/leads/route` fires synchronously. Evaluates routing rules in priority order (specialty → geography → round-robin). Assigns the lead to an LO. Writes `lead_assignments` row.

2. **SMS** (2–10 seconds): Twilio sends a personalized intro SMS from the LO's Twilio number (or branch pool number). Message generated by Claude Haiku 4.5 based on loan purpose. TCPA opt-out footer on every message. Consent source logged.

3. **Email** (2–10 seconds, parallel with SMS): Resend sends a personalized intro email from the LO's address. Subject line and body personalized by loan purpose and lead source.

4. **In-app push** (2–10 seconds, parallel): Real-time Supabase subscription pushes a notification to the LO's dashboard. Notification persists in the `lo_notifications` queue until acknowledged.

5. **Event logging**: Every action writes a `speed_to_lead_events` row with timestamp and metadata.

Total wall-clock target: **under 60 seconds from lead creation to SMS delivered**.

### 1.3 Escalation Ladder

The `speed-to-lead-escalation` Supabase Edge Function runs every 5 minutes via `pg_cron`. It scans `lead_assignments` for rows where `first_contact_at IS NULL` and `routing_status != 'contacted'`.

| Elapsed Since Assignment | Action |
|---|---|
| 5 minutes | Re-alert LO: repeat in-app push + SMS to LO's personal mobile: "You have an uncontacted lead — [borrower name] ([loan purpose]). Respond now." |
| 10 minutes | Alert team lead: in-app push + SMS to team lead. Dashboard banner on team lead's screen. |
| 30 minutes | Alert branch manager: in-app push + SMS + email to branch manager. Lead flagged `routing_status = 'escalated'`. Appears in branch manager dashboard with reassign button. |

Escalations **stop immediately** when LO calls `POST /api/leads/[id]/mark-contacted`.

### 1.4 Contact Marking

The LO marks contact by:
- Clicking "Mark Contacted" in the dashboard lead card
- Replying to the in-app notification
- Any subsequent outbound activity logged against the lead (call, SMS reply, email reply)

On mark-contacted:
- `lead_assignments.first_contact_at` is set to `NOW()`
- `lead_assignments.response_time_seconds` is calculated
- `lo_response_metrics` is updated (rolling average)
- Escalation stops for that lead

## SECTION 3 — API ROUTES

All routes live under `app/api/`. Use Next.js Route Handlers. All routes require Clerk authentication (`auth()` from `@clerk/nextjs/server`) except inbound webhooks (which use HMAC signature verification). All responses are typed with Zod.

### 3.1 `POST /api/leads/route`

Fires on every new lead. Called internally by the lead creation flow and by Supabase trigger.

**File:** `app/api/leads/route/route.ts`

**Request body:**
```typescript
{
  lead_id: string  // UUID of the newly created lead
}
```

**Logic:**
1. Fetch lead with `loan_purpose`, `source`, `branch_id`, `state` fields.
2. Load active routing rules for the org, ordered by `priority ASC`.
3. Evaluate rules in order:
   - **specialty**: if `loan_purpose` matches rule's `conditions.loan_types`, assign to `conditions.lo_id` (check LO is active + in business hours)
   - **geography**: if lead's state matches `conditions.states`, assign to that LO
   - **source**: if lead's source matches `conditions.sources`, assign to that LO
   - **round_robin**: fallback — fetch `round_robin_state`, advance index, assign next LO in pool (skip if on vacation/inactive)
4. Write `lead_assignments` row.
5. Update `leads.routing_status = 'assigned'`.
6. Log `routing_completed` event.
7. Trigger parallel: `sendIntroSMS()`, `sendIntroEmail()`, `pushLONotification()`.
8. Return `{ assignment_id, assigned_to_lo_id, method }`.

**Error handling:** If no rule matches and round-robin pool is empty, set `routing_status = 'unassigned'`, log `routing_failed`, alert branch manager immediately.

### 3.2 `POST /api/leads/[id]/mark-contacted`

Stops the escalation clock.

**File:** `app/api/leads/[id]/mark-contacted/route.ts`

**Request body:**
```typescript
{
  contact_method: 'sms' | 'call' | 'email' | 'in_person' | 'app'
  notes?: string
}
```

**Logic:**
1. Verify caller is the assigned LO or a manager in the same branch.
2. Fetch active `lead_assignments` row for this lead.
3. If `first_contact_at IS NOT NULL`, return 409 (already marked).
4. Set `first_contact_at = NOW()`, calculate `response_time_seconds`.
5. Update `leads.routing_status = 'contacted'`.
6. Log `contact_marked` event with `response_time_seconds` in metadata.
7. Trigger `updateLOResponseMetrics(lo_id)` (async, non-blocking).
8. Return `{ response_time_seconds, contact_method }`.

### 3.4 `POST /api/webhooks/zillow`

**File:** `app/api/webhooks/zillow/route.ts`

**Auth:** HMAC-SHA256 signature on `X-Zillow-Signature` header using `ZILLOW_WEBHOOK_SECRET` env var. Reject if invalid.

**Logic:**
1. Verify HMAC signature.
2. Parse Zillow lead payload (name, email, phone, property address, loan type).
3. Map Zillow loan type to internal `loan_purpose`.
4. Log TCPA consent: Zillow leads come pre-consented via Zillow's terms — set `tcpa_consent_given = TRUE`, `tcpa_consent_source = 'zillow_platform'`.
5. Insert into `leads` with `source = 'zillow'`.
6. Trigger `POST /api/leads/route` internally.
7. Return `200 OK` immediately (Zillow requires fast response).

**Zillow field mapping:**

### 3.5 `POST /api/leads/[id]/reassign`

Managers and team leads can reassign an uncontacted lead.

**File:** `app/api/leads/[id]/reassign/route.ts`

**Request body:**
```typescript
{
  new_lo_id: string,
  reason: 'capacity' | 'specialty' | 'geography' | 'escalation' | 'manual'
  notes?: string
}
```

**Logic:**
1. Verify caller is team_lead, branch_manager, or admin.
2. Mark current active assignment `is_active = FALSE`.
3. Create new `lead_assignments` row with `assignment_method = 'manual_reassign'`.
4. Update `leads.routing_status = 'assigned'`.
5. Trigger SMS/email/push to NEW LO (same intro sequence, no duplicate to borrower).
6. Log `lead_reassigned` event.
7. Return `{ new_assignment_id }`.

## SECTION 4 — EDGE FUNCTION: `speed-to-lead-escalation`

**File:** `supabase/functions/speed-to-lead-escalation/index.ts`

Scheduled via Supabase `pg_cron` every 5 minutes:
```sql
SELECT cron.schedule(
  'speed-to-lead-escalation',
  '*/5 * * * *',
  $$SELECT net.http_post(
    url := current_setting('app.supabase_functions_url') || '/speed-to-lead-escalation',
    headers := jsonb_build_object(
      'Authorization', 'Bearer ' || current_setting('app.service_role_key'),
      'Content-Type', 'application/json'
    ),
    body := '{}'::jsonb
  )$$
);
```

**Full function logic:**

```typescript
import { createClient } from '@supabase/supabase-js'
import { Twilio } from 'twilio'

const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
)

const twilio = new Twilio(
  Deno.env.get('TWILIO_ACCOUNT_SID')!,
  Deno.env.get('TWILIO_AUTH_TOKEN')!
)

Deno.serve(async () => {
  const now = new Date()

  // Fetch all uncontacted active assignments
  const { data: assignments } = await supabase
    .from('lead_assignments')
    .select(`
      *,
      leads(id, first_name, last_name, loan_purpose, branch_id),
      lo:assigned_to(id, first_name, last_name, phone, email),
      org:org_id(id, name)
    `)
    .is('first_contact_at', null)
    .eq('is_active', true)
    .lt('assigned_at', new Date(now.getTime() - 4.5 * 60 * 1000).toISOString())

  for (const assignment of assignments ?? []) {
    const minutesElapsed = (now.getTime() - new Date(assignment.assigned_at).getTime()) / 60000

    // Determine which escalation tier applies
    let targetTier: 1 | 2 | 3 | null = null

    if (minutesElapsed >= 30 && assignment.escalation_level < 3) {
      targetTier = 3
    } else if (minutesElapsed >= 10 && assignment.escalation_level < 2) {
      targetTier = 2
    } else if (minutesElapsed >= 5 && assignment.escalation_level < 1) {
      targetTier = 1
    }

    if (!targetTier) continue

    await fireEscalation(assignment, targetTier, minutesElapsed)
  }

  return new Response(JSON.stringify({ processed: assignments?.length ?? 0 }))
})

async function fireEscalation(assignment: any, tier: 1 | 2 | 3, minutesElapsed: number) {
  const borrowerName = `${assignment.leads.first_name} ${assignment.leads.last_name}`
  const loanPurpose = assignment.leads.loan_purpose ?? 'unknown'

  if (tier === 1) {
    // Re-alert the LO
    await sendEscalationSMS(
      assignment.lo.phone,
      `AshleyIQ ALERT: ${borrowerName} (${loanPurpose}) has been waiting ${Math.round(minutesElapsed)} minutes for your response. Contact them now before they go to another lender.`
    )
    await insertPushNotification(assignment.assigned_to, assignment.lead_id, 'escalation_5min', {
      borrower_name: borrowerName,
      minutes_elapsed: Math.round(minutesElapsed),
    })
  }

  if (tier === 2) {
    // Alert team lead
    const teamLead = await getTeamLead(assignment.branch_id)
    if (teamLead) {
      await sendEscalationSMS(
        teamLead.phone,
        `TEAM LEAD ALERT: ${assignment.lo.first_name} ${assignment.lo.last_name} has not contacted ${borrowerName} (${loanPurpose}) in ${Math.round(minutesElapsed)} minutes. Login to AshleyIQ to reassign.`
      )
      await insertPushNotification(teamLead.id, assignment.lead_id, 'escalation_10min', {
        lo_name: `${assignment.lo.first_name} ${assignment.lo.last_name}`,
        borrower_name: borrowerName,
        minutes_elapsed: Math.round(minutesElapsed),
      })
    }
  }

  if (tier === 3) {
    // Alert branch manager
    const branchManager = await getBranchManager(assignment.branch_id)
    if (branchManager) {
      await sendEscalationSMS(
        branchManager.phone,
        `BRANCH MANAGER ALERT: Lead ${borrowerName} (${loanPurpose}) has been uncontacted for ${Math.round(minutesElapsed)} minutes. Assigned to ${assignment.lo.first_name} ${assignment.lo.last_name}. Reassign immediately in AshleyIQ.`
      )
      await sendEscalationEmail(
        branchManager.email,
        assignment.lead_id,
        borrowerName,
        assignment.lo,
        minutesElapsed
      )
      await insertPushNotification(branchManager.id, assignment.lead_id, 'escalation_30min', {
        lo_name: `${assignment.lo.first_name} ${assignment.lo.last_name}`,
        borrower_name: borrowerName,
        minutes_elapsed: Math.round(minutesElapsed),
      })
    }

    // Flag lead as escalated
    await supabase
      .from('leads')
      .update({ routing_status: 'escalated' })
      .eq('id', assignment.lead_id)
  }

  // Update escalation level
  await supabase
    .from('lead_assignments')
    .update({
      escalation_level: tier,
      last_escalation_at: new Date().toISOString(),
    })
    .eq('id', assignment.id)

  // Log event
  await supabase.from('speed_to_lead_events').insert({
    lead_id: assignment.lead_id,
    assignment_id: assignment.id,
    org_id: assignment.org_id,
    event_type: `escalation_${tier === 1 ? '5min' : tier === 2 ? '10min' : '30min'}`,
    metadata: {
      lo_id: assignment.assigned_to,
      minutes_elapsed: Math.round(minutesElapsed),
      alerted_via: tier === 3 ? ['push', 'sms', 'email'] : ['push', 'sms'],
    },
    elapsed_seconds_from_lead: Math.round(minutesElapsed * 60),
  })
}
```

## SECTION 5 — SMS TEMPLATES (Claude Haiku 4.5)

**File:** `lib/speed-to-lead/sms-templates.ts`

All SMS messages are generated by Claude Haiku 4.5. The LO's name, the borrower's first name, and the loan purpose are injected into the prompt. Every message includes a TCPA opt-out footer. Every message logs the consent source on the lead record before sending.

### 5.1 TCPA Compliance — Non-Negotiable

Before sending ANY SMS to a lead:

1. Verify `leads.tcpa_consent_given = TRUE`. If FALSE, do NOT send SMS — fall back to email only and flag for manual review.
2. Log the consent source in `leads.tcpa_consent_source`.
3. Every outbound SMS must end with: `\n\nReply STOP to opt out.`
4. On receipt of `STOP` reply from Twilio webhook: set `tcpa_consent_given = FALSE`, log `tcpa_optout_received` event, never send to that number again.

### 5.2 Claude Haiku Prompt for SMS Generation

```typescript
import Anthropic from '@anthropic-ai/sdk'

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY })

export async function generateIntroSMS(params: {
  lo_first_name: string
  lo_last_name: string
  borrower_first_name: string
  loan_purpose: 'purchase' | 'refinance' | 'cashout_refi' | 'heloc' | 'construction' | 'unknown'
  source: string
  property_state?: string
}): Promise<string> {

  const purposeContext: Record<string, string> = {
    purchase: 'buying a home',
    refinance: 'refinancing their mortgage',
    cashout_refi: 'doing a cash-out refinance',
    heloc: 'exploring a home equity line of credit',
    construction: 'financing a construction project',
    unknown: 'exploring mortgage options',
  }

  const message = await anthropic.messages.create({
    model: 'claude-haiku-4-5',
    max_tokens: 200,
    messages: [{
      role: 'user',
      content: `Write a brief, warm, professional intro SMS from a mortgage loan officer to a new lead.

LO name: ${params.lo_first_name} ${params.lo_last_name}
Borrower first name: ${params.borrower_first_name}
Loan purpose: ${purposeContext[params.loan_purpose]}
${params.property_state ? `State: ${params.property_state}` : ''}

Requirements:
- Address the borrower by first name
- Introduce the LO by first name only (not full name — too formal for SMS)
- Reference the loan purpose naturally
- Keep it under 160 characters BEFORE the opt-out footer
- Warm but professional — not salesy
- End with an open question that invites a reply
- Do NOT include the opt-out footer (it will be appended automatically)
- Do NOT use emojis
- Do NOT use exclamation points more than once

Return ONLY the SMS body text, nothing else.`
    }]
  })

  const body = (message.content[0] as { type: 'text'; text: string }).text.trim()
  return `${body}\n\nReply STOP to opt out.`
}
```

### 5.4 SMS Sending Function

**File:** `lib/speed-to-lead/send-intro-sms.ts`

```typescript
import twilio from 'twilio'
import { generateIntroSMS } from './sms-templates'
import { createServiceClient } from '@/lib/supabase/service'

const client = twilio(
  process.env.TWILIO_ACCOUNT_SID!,
  process.env.TWILIO_AUTH_TOKEN!
)

export async function sendIntroSMS(leadId: string, loId: string) {
  const supabase = createServiceClient()

  // Fetch lead and LO data
  const { data: lead } = await supabase
    .from('leads')
    .select('first_name, last_name, phone, loan_purpose, source, state, tcpa_consent_given, tcpa_consent_source, branch_id')
    .eq('id', leadId)
    .single()

  if (!lead) throw new Error(`Lead ${leadId} not found`)

  // TCPA gate — never bypass
  if (!lead.tcpa_consent_given) {
    await supabase.from('speed_to_lead_events').insert({
      lead_id: leadId,
      org_id: lead.branch_id, // TODO: resolve org_id properly
      event_type: 'sms_failed',
      metadata: { reason: 'no_tcpa_consent', tcpa_consent_source: lead.tcpa_consent_source ?? null },
    })
    return { skipped: true, reason: 'no_tcpa_consent' }
  }

  const { data: lo } = await supabase
    .from('profiles')
    .select('first_name, last_name, twilio_number, org_id')
    .eq('id', loId)
    .single()

  if (!lo?.twilio_number) {
    // Fall back to branch pool number
    // TODO: fetch branch Twilio number
    throw new Error(`LO ${loId} has no Twilio number configured`)
  }

  // Generate personalized message
  const body = await generateIntroSMS({
    lo_first_name: lo.first_name,
    lo_last_name: lo.last_name,
    borrower_first_name: lead.first_name,
    loan_purpose: lead.loan_purpose ?? 'unknown',
    source: lead.source,
    property_state: lead.state ?? undefined,
  })

  // Log queued
  await supabase.from('speed_to_lead_events').insert({
    lead_id: leadId,
    org_id: lo.org_id,
    event_type: 'sms_queued',
    metadata: { to_number: lead.phone, from_number: lo.twilio_number },
  })

  // Send via Twilio
  const message = await client.messages.create({
    to: lead.phone,
    from: lo.twilio_number,
    body,
    statusCallback: `${process.env.NEXT_PUBLIC_APP_URL}/api/webhooks/twilio/status`,
  })

  // Log sent
  await supabase.from('speed_to_lead_events').insert({
    lead_id: leadId,
    org_id: lo.org_id,
    event_type: 'sms_sent',
    metadata: { twilio_sid: message.sid, status: message.status, template: lead.loan_purpose ?? 'unknown' },
  })

  return { twilio_sid: message.sid }
}
```
